1. final 키워드
   변수 , 메소드 , 클래스에 붙일 수 있는 키워드 이다. 
   "read only" 의 개념으로 보면 된다 .(편집 불가)

  1) 변수 : 편집이 불가능한 데이터
    	 상수(constant)
             수학의 pi

  2) 메소드 : 서브 클래스로 하여금 오버라이딩을 못하게 하는 메소드
                대신 , 이 메소드는 완벽하게 구현해야한다. 
                (못쓰게 했으면 그에 대한 책임이 있어야함)

  3) 클래스 : final 클래스는 하위 클래스를 만들 수 없다.(서브클래싱 불가)
	    대신 , 이 클래스 내의 모든 기능들은 완벽하게 구현해 줘야 한다.
                (불안정한 상태에서 final 을 붙이면 안됨)
	    String , Integer 클래스가 대표적 final 클래스이다.

3. 추상(abstract)메소드
   메소드 이름 앞에 abstarct 키워드를 붙여 주는 메소드
   내용물이 존재하지 않는 형태의 모델 개념으로, 불완전한 메소드 이다.
   

   1) 특징 
	-추상 메소드를 포함하는 클래스는 반드시 추상 클래스이어야 한다.
	-실제 구현할 내용이 없으므로 메소드 body(중괄호) 대신 세미콜론을 붙여 줘야 한다.
  	 (ex. 도형을 만들어라(추상적) → 어떤도형?(구체적으로 들어옴) 
	      추상 메소드는 구체적으로 명시할수 없음 , 움직여라 ...어떻게 움직이라는지는 명시 X)
	- 추상 클래스를 상속 받는 클래스는 반드시 추상 메소드를 오버라이딩 해야된다.
            - 스스로 만들어 내는 자생력이 없다.(cannot instance...)
            - new 연산자는 사용이 불가능하다.
	- 다만 , 승급 시 수퍼 클래스로 사용 가능하다.
	- 자바에서 대표적 추상 클래스는 Canlendar  클래스이다 
	- 추상 클래스 내에는 일반 변수와 메소드도 사용 가능하다.
	- 추상 클래스를 상속 받는 추상 클래스는 오버라이딩 하지 않아도 된다.


	[상속관계]
  	A클래스 : 추상메소드 4개

            B클래스 : 추상메소드 1개 -> 오버라이딩 X , 왜? 불완전한 상태를 오버라이딩해봤자 손해...

            C클래스 : 추상메소드 0개 -> 오버라이딩 5개
	
   2) 목적 / 이유
	- 업무 규칙을 위한 강제적인 표준 규범을 잡기 위하여 사용한다.
	   (각자에 맞는 방법으로 코딩하기 때문에 관리하기 어려움)

4. 인터페이스
   자바에서 지원되지 않는 다중 상속을 충족시키기 위하여 도입한 개념 

   1) 문법
	class 서브 클래스 extends 수퍼 클래스 implements 인터페이스1, 인터페이스2 

   2) 특징
	상수(constant)와 추상 메소드로만 구성이 되어 있습니다.
	상속을 받고자 하는 경우 implements 키워드를 사용하면 된다.
	인터페이스 끼리의 상속은 extends 를 사용한다.
	(같은 부류 : extends , 다른 부류 : implements )
	인터페이스 간의 상속에서 하위 인터페이스는 상위 인터페이스의 모든 메소드를 오버라이딩 하지 않는다.
	

   3) 변수
	모든 변수는 암시적으로 public static final 키워드가 숨겨져 있다. 
	final 변수는 반드시 초기 값을 부여 해야 한다.
	(final은 불변의 진리값 , 편집이 불가능하므로 값이 안들어있음 )
   	static 변수이므로 [인터페이스 이름 .변수이름]으로 접근이 가능하다
  	(클래스에서 static 변수는 클래스 변수라고 의미하며 확장개념로 인터페이스 변수라고 봐야한다)

   4) 메소드 
	모든 메소드는 암시적으로 public abstract라는 키워드가 숨겨져 있다.

   4) 사용처
	컬렉션(자료 구조)
	자바의 GUI(스윙 , JAVA fx)
	스마트 폰 개발(안드로이드)
	자바 프레임 워크  과정 
	웹 프로그래밍 과정
