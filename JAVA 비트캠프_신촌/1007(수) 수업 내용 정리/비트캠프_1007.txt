1. 생성자 
   객체가 heap 에 생성된 다음 자바 시스템에 자동으로 1번만 호출되는 특수한 메소드 

    1)특징 
	생성자 오버로딩(매개변수의 타입 , 갯수가 다를때 서로 생성자 오버로딩 된다.)
	생성자는 개발자가 직접 정의할 수 있다.
	개발자가 직접 정의하게 되면 기본생성자(눈에 보이지 않던 생성자)는 사라진다.
	반환되지 않고 상속되지 않습니다.
	this() : 매개변수가 없는 생성자 호출 


    2)목적
	인스턴스 변수의 값을 초기화 할 용도로 사용된다.
	생성되자 마자 무엇인가를 구현하고 자 할 때 사용된다.
 
    3)기본 생성자
	개발자의 눈에 보이지 않고 , 매개 변수가 0개인 생성자를 의미한다.
	(눈에 안보이는데 인스턴스 변수 값을 어떻게 초기화 할 것인가?
	 → 개발자가 직접 만들면 됨)

    4)★작성 방법 
	이름 : 클래스의 이름과 반드시 동일해야 한다.
	매개변수	: 개발자가 임의로 지정하면 된다.
	반환타입 : 명시하지 않는다.
